You are absolutely correct. To handle multiple formats (ISO 19115 XML, JSON, RDF/Turtle) and ensure the system is extensible (easy to add new formats later) and reusable, we must apply the Strategy Pattern combined with the Factory Pattern.

This approach allows your ETL logic to remain "closed for modification but open for extension" (Open/Closed Principle), which is a key clean architecture requirement.
